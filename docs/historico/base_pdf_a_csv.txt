Instrucciones y código para convertir exámenes PDF a CSV
Plantilla operativa para proyecto de conversión PDF → CSV
Objetivo: convertir exámenes PDF a CSV de forma literal, sin responder las preguntas, sin inventar contenido y tomando las respuestas exclusivamente de la plantilla del propio PDF o de la lista aportada por el usuario.
1. Reglas obligatorias de conversión
•	Extraer literalmente el enunciado completo de cada pregunta y las cuatro opciones a), b), c), d).
•	Incluir todas las preguntas del examen y todas las preguntas de reserva si existen.
•	No contestar preguntas por conocimiento propio. La columna correcta debe salir solo de la plantilla/solucionario del PDF o de la lista que aporte el usuario.
•	No resumir, no reformular y no acortar enunciados ni opciones.
•	Si falta la respuesta de una pregunta en la plantilla, dejar el campo correcta en blanco: "".
•	Si una pregunta está anulada, conservarla en el CSV y reflejar literalmente que está anulada o que se sustituye por una reserva, si así aparece en el PDF.
•	Mantener el bloque/materia exactamente como lo indique el usuario.
2. Formatos CSV habituales
•	Con número de pregunta: numero,pregunta,a,b,c,d,correcta,bloque
•	Sin número de pregunta: pregunta,a,b,c,d,correcta,bloque
•	Examen común de abogacía si el usuario lo pide así: pregunta,a,b,c,d,correcta,materia
•	Todos los campos deben ir entre comillas dobles mediante csv.QUOTE_ALL.
3. Validación obligatoria antes de entregar
•	Comprobar que el número de filas útiles coincide con el examen: por ejemplo 20+2 reservas = 22; 25+2 reservas = 27; 50+reservas en común.
•	Comprobar que todas las filas tienen el mismo número de columnas que la cabecera.
•	Comprobar que no hay cabeceras repetidas dentro del archivo.
•	Comprobar que no hay filas vacías finales.
•	Comprobar que las respuestas coinciden con la plantilla de la última hoja.
•	Comprobar que no se han inventado preguntas, opciones ni respuestas.
4. Criterio para PDFs con OCR difícil
•	Si el texto extraído está desordenado por columnas, comprobar visualmente el PDF o sus imágenes renderizadas.
•	No completar por intuición si una línea no se lee bien.
•	Si no puede verificarse una parte, dejar constancia de qué pregunta u opción no es verificable.
5. Prompt base para usar en el proyecto
Convierte los exámenes a CSV como antes, con: texto completo y literal del PDF; preguntas [rango], incluidas reservas si hubiera; respuestas tomadas de la plantilla del propio PDF en la última hoja; campos entre comillas; cabecera: numero,pregunta,a,b,c,d,correcta,bloque. Si una pregunta no tiene respuesta en la plantilla, deja el campo correcta en blanco "". No respondas las preguntas ni inventes respuestas. Revisa antes de entregar que todas las filas tengan las columnas correctas, que no haya cabeceras repetidas, que no haya filas vacías y que las respuestas coincidan con la plantilla.
6. Flujo de trabajo usado
Paso	Acción
Paso 1	Leer el PDF y extraer el texto por páginas. Si el PDF viene como imagen o con columnas, usar renderizado/OCR o revisión visual.
Paso 2	Separar el bloque de preguntas del bloque de plantilla de respuestas.
Paso 3	Extraer cada pregunta con su número, enunciado y opciones a, b, c, d.
Paso 4	Extraer la plantilla final de respuestas: número → letra. Ejemplo: 1 B, 2 D, 3 A.
Paso 5	Construir filas CSV aplicando la respuesta que corresponda a cada número.
Paso 6	Escribir el CSV con csv.writer, quoting=csv.QUOTE_ALL y codificación utf-8-sig.
Paso 7	Validar estructura, filas, columnas, cabecera, filas vacías y respuestas.
7. Código Python base
Este script es una base reutilizable. La extracción automática funciona mejor en PDFs con texto claro. En PDFs escaneados o con columnas, se debe revisar visualmente y corregir manualmente los bloques que el extractor mezcle.
import csv
import re
from pathlib import Path
 
# Opcional para PDFs con capa de texto:
# pip install pymupdf
try:
    import fitz  # PyMuPDF
except ImportError:
    fitz = None
 
 
def extract_text_from_pdf(pdf_path: str) -> str:
    """Extrae texto de un PDF con PyMuPDF.
    Si el PDF es escaneado o el texto sale desordenado por columnas, revisar visualmente.
    """
    if fitz is None:
        raise RuntimeError("Instala pymupdf: pip install pymupdf")
 
    doc = fitz.open(pdf_path)
    pages = []
    for i, page in enumerate(doc, start=1):
        text = page.get_text("text")
        pages.append(f"\n--- PAGE {i} ---\n{text}")
    return "\n".join(pages)
 
 
def clean_text(text: str) -> str:
    """Limpieza mínima. No reformula ni resume."""
    text = text.replace("\r", "\n")
    text = re.sub(r"[ \t]+", " ", text)
    text = re.sub(r"\n{3,}", "\n\n", text)
    return text.strip()
 
 
def parse_answer_key(text: str) -> dict[int, str]:
    """Extrae respuestas tipo:
    1 B
    2 D
    3 A
    También conserva valores dudosos como D? o ANULADA.
    """
    answers = {}
    pattern = re.compile(r"(?m)^\s*(\d{1,3})\s*[\.\-:]?\s*([A-Da-d](?:\?)?|ANULADA|anulada|/)\s*$")
    for match in pattern.finditer(text):
        num = int(match.group(1))
        ans = match.group(2).strip()
        answers[num] = ans
    return answers
 
 
def normalize_option_label(label: str) -> str:
    return label.strip().lower().replace(")", "").replace(".", "")
 
 
def parse_questions_basic(text: str) -> list[dict]:
    """Parser básico para preguntas tipo:
    1.- Enunciado...
    a) opción
    b) opción
    c) opción
    d) opción
 
    En PDFs con columnas o OCR irregular puede requerir ajuste manual.
    """
    # Normalizar etiquetas de opciones a formato a) b) c) d)
    t = text
    t = re.sub(r"(?m)^\s*([abcdABCD])\s*[\.)]\s*", lambda m: f"\n{m.group(1).lower()}) ", t)
 
    # Detectar inicios de preguntas: 1.-, 1., 1 -, 1 –
    q_start = re.compile(r"(?m)^\s*(\d{1,3})\s*(?:[\.\-–]|\.-)\s*")
    matches = list(q_start.finditer(t))
    questions = []
 
    for idx, m in enumerate(matches):
        num = int(m.group(1))
        start = m.end()
        end = matches[idx + 1].start() if idx + 1 < len(matches) else len(t)
        block = t[start:end].strip()
 
        # Cortar si llega a plantilla de respuestas
        block = re.split(r"(?i)\n\s*(JUNIO|SEPT|SEPTIEMBRE)\s+\d{4}", block)[0].strip()
 
        opt_pattern = re.compile(r"\n\s*([abcd])\)\s*")
        opt_matches = list(opt_pattern.finditer("\n" + block))
        if len(opt_matches) < 4:
            # No inventar: guardar bloque completo para revisión.
            questions.append({
                "numero": num,
                "pregunta": block,
                "a": "",
                "b": "",
                "c": "",
                "d": "",
                "parse_ok": False,
            })
            continue
 
        block2 = "\n" + block
        question_text = block2[:opt_matches[0].start()].strip()
        opts = {}
        for j, om in enumerate(opt_matches[:4]):
            label = om.group(1)
            o_start = om.end()
            o_end = opt_matches[j + 1].start() if j + 1 < 4 else len(block2)
            opts[label] = block2[o_start:o_end].strip()
 
        questions.append({
            "numero": num,
            "pregunta": question_text,
            "a": opts.get("a", ""),
            "b": opts.get("b", ""),
            "c": opts.get("c", ""),
            "d": opts.get("d", ""),
            "parse_ok": True,
        })
 
    return questions
 
 
def build_rows(questions: list[dict], answers: dict[int, str], bloque: str, include_numero: bool = True) -> list[list[str]]:
    rows = []
    for q in questions:
        num = q["numero"]
        correcta = answers.get(num, "")  # si no existe respuesta, queda en blanco
        if include_numero:
            rows.append([
                str(num),
                q["pregunta"],
                q["a"],
                q["b"],
                q["c"],
                q["d"],
                correcta,
                bloque,
            ])
        else:
            rows.append([
                q["pregunta"],
                q["a"],
                q["b"],
                q["c"],
                q["d"],
                correcta,
                bloque,
            ])
    return rows
 
 
def write_csv(rows: list[list[str]], output_path: str, include_numero: bool = True) -> None:
    header = ["numero", "pregunta", "a", "b", "c", "d", "correcta", "bloque"] if include_numero else ["pregunta", "a", "b", "c", "d", "correcta", "bloque"]
    with open(output_path, "w", newline="", encoding="utf-8-sig") as f:
        writer = csv.writer(
            f,
            delimiter=",",
            quotechar='"',
            quoting=csv.QUOTE_ALL,
            lineterminator="\n",
        )
        writer.writerow(header)
        writer.writerows(rows)
 
 
def validate_csv(path: str, expected_columns: int, expected_rows: int | None = None) -> list[str]:
    errors = []
    with open(path, "r", encoding="utf-8-sig", newline="") as f:
        rows = list(csv.reader(f))
 
    if not rows:
        return ["CSV vacío"]
 
    header = rows[0]
    if len(header) != expected_columns:
        errors.append(f"Cabecera con {len(header)} columnas; esperadas {expected_columns}")
 
    data_rows = rows[1:]
    if expected_rows is not None and len(data_rows) != expected_rows:
        errors.append(f"Filas útiles: {len(data_rows)}; esperadas {expected_rows}")
 
    for line_number, row in enumerate(data_rows, start=2):
        if not any(cell.strip() for cell in row):
            errors.append(f"Fila vacía en línea {line_number}")
        if len(row) != expected_columns:
            errors.append(f"Fila {line_number} con {len(row)} columnas; esperadas {expected_columns}")
        if row == header:
            errors.append(f"Cabecera repetida en línea {line_number}")
 
    return errors
 
 
def convert_pdf_to_csv(pdf_path: str, output_csv: str, bloque: str, expected_rows: int | None = None, include_numero: bool = True):
    raw = extract_text_from_pdf(pdf_path)
    text = clean_text(raw)
    answers = parse_answer_key(text)
    questions = parse_questions_basic(text)
 
    # Limitar a expected_rows si el usuario lo indica y el extractor ha capturado ruido posterior.
    if expected_rows is not None:
        questions = [q for q in questions if 1 <= q["numero"] <= expected_rows]
 
    # Avisar de preguntas mal separadas.
    bad = [q["numero"] for q in questions if not q.get("parse_ok")]
    if bad:
        print(f"Revisar manualmente preguntas mal separadas: {bad}")
 
    rows = build_rows(questions, answers, bloque, include_numero=include_numero)
    write_csv(rows, output_csv, include_numero=include_numero)
 
    expected_columns = 8 if include_numero else 7
    errors = validate_csv(output_csv, expected_columns=expected_columns, expected_rows=expected_rows)
    if errors:
        print("Errores de validación:")
        for e in errors:
            print("-", e)
    else:
        print("CSV validado correctamente")
 
 
if __name__ == "__main__":
    convert_pdf_to_csv(
        pdf_path="examen.pdf",
        output_csv="examen.csv",
        bloque="Medicina legal y forense",
        expected_rows=27,      # ejemplo: 25 preguntas + 2 reservas
        include_numero=True,   # cabecera: numero,pregunta,a,b,c,d,correcta,bloque
    )
8. Plantilla manual cuando el PDF tiene columnas o OCR irregular
En muchos exámenes, especialmente los de Medicina legal y forense con dos columnas, el extractor puede mezclar preguntas. En ese caso, se usa esta plantilla manual dentro del mismo script para garantizar literalidad:
import csv
 
rows = [

        "Opción B literal",
        "Opción C literal",
        "Opción D literal",
        "B",

    # Repetir hasta incluir todas las preguntas y reservas.

9. Script independiente de validación
import csv
from pathlib import Path
 
def validate_csv_file(path, expected_columns=8, expected_rows=None):
    with open(path, "r", encoding="utf-8-sig", newline="") as f:
        rows = list(csv.reader(f))
 
    errors = []
    if not rows:
        return ["CSV vacío"]
 
    header = rows[0]
    data = rows[1:]
 
    if len(header) != expected_columns:
        errors.append(f"Cabecera con {len(header)} columnas; esperadas {expected_columns}")
 
    if expected_rows is not None and len(data) != expected_rows:
        errors.append(f"Número de filas útiles incorrecto: {len(data)}; esperadas {expected_rows}")
 
    for i, row in enumerate(data, start=2):
        if row == header:
            errors.append(f"Cabecera repetida en línea {i}")
        if not any(cell.strip() for cell in row):
            errors.append(f"Fila vacía en línea {i}")
        if len(row) != expected_columns:
            errors.append(f"Fila {i}: {len(row)} columnas; esperadas {expected_columns}")
 
    return errors
 
if __name__ == "__main__":
    csv_path = "salida.csv"
    errors = validate_csv_file(csv_path, expected_columns=8, expected_rows=27)
    if errors:
        print("ERRORES")
        for e in errors:
            print("-", e)
    else:
        print("OK: CSV válido")
10. Checklist final antes de entregar
•	El CSV abre correctamente en Excel/LibreOffice y también se puede leer con csv.reader.
•	La cabecera coincide exactamente con la pedida por el usuario.
•	Todas las preguntas y reservas están incluidas.
•	La columna correcta coincide con la plantilla del PDF.
•	Las preguntas sin respuesta en plantilla tienen correcta = "".
•	No hay cabeceras repetidas.
•	No hay filas vacías al final.
•	No se ha respondido ninguna pregunta por cuenta propia.
11. Errores típicos que hay que evitar
•	Copiar una pregunta reconstruida en vez de literal del PDF.
•	Olvidar las preguntas de reserva.
•	Poner una segunda cabecera en mitad del CSV.
•	Responder jurídicamente por memoria en vez de usar la plantilla.
•	Cambiar el bloque indicado por el usuario.
•	Dejar opciones sin comillas cuando contienen comas o punto y coma.
•	Confundir examen junio A con junio B o septiembre.
